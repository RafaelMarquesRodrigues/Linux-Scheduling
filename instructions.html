Instruções para começar a simulação do escalonamento:

    1. No menu clique em iniciar para entrar no simulador de escalonamento Linux;

    2. Ao entrar no simulador, primeiro selecione a quantidade de processos que você deseja simular o escalonamento;

    3. A próxima etapa é escolher como os passos ocorrerão: passo-a-passo ou automaticamente. Quando selecionado Automaticamente os passos serão executados sem pausa, e quando selecionado Passo-a-passo é preciso apertar o botão X para prosseguir;

    4. Após escolher o número de processos e como eles serão simulados é possível optar por duas opções:
      -Preencher as especificações dos processos:
          A prioridade, o tempo necessário para completar a execução, o tipo de escalonamento (FIFO ou Round-Robin), o tempo necessário de execução de E/S, a chance do processo requisitar E/S e a chance do processo deixar a CPU.
          Após isso, basta clicar no botão Pronto para iniciar a simulação.

      -Preenchemento automático das especificações de cada processo:
          Para isso acontecer você pode optar por 3 modos:
            -O modo Aleatório pega o número de processos escolhidos e preenche cada especificação de forma aleatória;
            -O modo CPU Bounded preenche as especificações de modo que tenham baixa solicitação de E/S e baixo tempo de execução destes.
            -O modo I/O Bounded preenche as especificações de modo que tenham alta solicitação de E/S e com alto tempo de execução destes.

    5. Pronto! Agora a simulação foi iniciada. Vamos explicar umas funções extras na execução:
      -No painel a direita será exibido a descrição dos eventos conforme vão acontecendo;
      -Selecione o botão Log para verificar todas as descrições que já foram exibidas;
      -Selecione o botão Estatisticas para exibir alguns dados da CPU;
      -Selecione o botão Ajuda caso tenha alguma dúvida no decorrer da execução;

Instructions to start the simulation of the scheduling :

    1. At menu, click Start to enter the Linux Scheduling Simulator;

    2. Entering the simulator, first choose a quantity of processes that you want to simulate the scheduling;

    3. The next phase is choose how the steps will occur: Step-by-step or automatically. When the option Automatic is chosen the steps will be executed without interruptions, and when the option Step-by-step is chosen is needed to press the arrow button to continue.

    4. After choose the number of processes and how tey will be simulated is possible choose two options:
      -Fill the processes specifications:
          The priority, the necessary time to complete the execution, the scheduling type (FIFO or Round-Robin), the necessary time to execute an I/O request, the process chance to requisite ana I/O request and the process chance to leave the CPU.
          After that, click on the "ready" button to start the simulation.

      -Automatic fill of each process specification:
        Here we have three modes:
          -The Aleatory one fill random the processes specifications.
          -The CPU Bounded one fill the specifications with low I/O requests and low execution time.
          -The I/O Bounded one fill the specifications with high I/O requests and high execution time.

    5. Done! Now the simulation was started. Let us explain some extra functions in execution:
      -In the left painel will be shown the events description while they will occur;
      -Select the Log button to verify all the descriptions that were shown;
      -Select the Statistics button to show some CPU data;
      -Select the Help button case anything is unclear over the execution.
